Yeh task **Shodan** ke baare mein hai. Aasan lafzon mein samjhein toh jaise Google aam websites ke pages ko search karta hai, **Shodan internet se connect hone wali har "Device" ko search karta hai.** Isi liye isko **"Search Engine for the Internet of Things (IoT)"** kaha jata hai. Yeh har waqt poore internet ko scan karta rehta hai aur check karta hai ke kahan kaun sa router, traffic camera, server, ya industrial system chal raha hai.

Chalein isko points mein break karte hain ke yeh kaise kaam karta hai aur is task mein kya bataya gaya hai:

### 1. Banner Grabbing (Information Nikalna)
Jab Shodan kisi device ko scan karta hai, toh woh us device se milne wale response (headers) ko save kar leta hai.
 * **Example:** Agar aap Shodan par apache 2.4.1 search karenge, toh internet par jitne bhi servers is specific version ko chala rahe hain, Shodan unki list nikal ke de dega.

 * **Security Use:** Penetration testing ke doran agar aapko pata chal jaye ke target company Apache 2.4.1 use kar rahi hai, aur aapko maloom ho ke is version mein koi specific **CVE (vulnerability)** hai, toh aapka kaam bohot aasan ho jata hai.

### 2. Shodan Query Filters (Sahi Target Dhoondna)
Shodan par aam search karne se lakhon results aayenge. Is liye specific results ke liye **Filters** use hote hain:
| Filter | Kaam Kya Hai? | Example |
|---|---|---|
| **country:** | Sirf kisi aik mulk ke results dekhne ke liye (e.g., IE = Ireland, PK = Pakistan). | country:IE |
| **port:** | Kisi specific port par chalne wali devices dhoondna (e.g., SSH ke liye port 22). | port:22 |
| **org:** | Kisi specific company ya organization ke IP addresses dhoondna. | org:"Amazon Web Services" |
| **hostname:** | Kisi specific domain ya sub-domain ko target karna. | hostname:fakebank.thm |
Aap inko mila kar bhi likh sakte hain, jaise: apache 2.4.1 country:PK port:80 (Iska matlab hoga Pakistan mein chalne wale woh servers jo port 80 par Apache 2.4.1 use kar rahe hain).
