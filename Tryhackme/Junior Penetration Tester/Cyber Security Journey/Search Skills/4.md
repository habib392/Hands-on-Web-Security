Yeh task pure cyber security industry ke sab se important concepts mein se aik ke baare mein hai: **CVE (Common Vulnerabilities and Exposures)** aur **CVSS (Common Vulnerability Scoring System)**.
Aap Web Penetration Testing aur PortSwigger ke labs solve karte hain, toh aapko pata hona chahiye ke jab koi vulnerability dunya mein pehli baar milti hai, toh usko handle kaise kiya jata hai. Chalein isko aasan alfaz mein break down
karte hain:

### 1. CVE Kya Hai? (Universal Dictionary)
Dunya mein hazaron softwares hain aur un mein roz naye bugs aate hain. Agar har banda aik hi bug ko alag naam se pukarega toh confusion ho jayegi. Is liye **CVE** banaya gaya, jo ke dunya ki tamam jani-pehchani vulnerabilities ki aik **universal dictionary** (reference point) hai.
Har vulnerability ko aik unique ID milti hai jiska format yeh hota hai:
CVE - [SAAL] - [NUMBER]
 * **Example:** CVE-2026-1337 (Iska matlab yeh vulnerability saal 2026 mein register hui hai aur iska number 1337 hai).
Kuch bohot badi aur khatarnak vulnerabilities ko aam naam bhi diye jaate hain (monikers), jaise **Log4Shell** ya **Heartbleed**.

### 2. CVSS Kya Hai? (Khatre Ka Score)
Har vulnerability aik jaisi khatarnak nahi hoti. Kisi se sirf website ka naam pata chalta hai, toh kisi se poora server hacker ke kabze mein aa jata hai. Is khatre ko naapne ke liye **CVSS Score** use hota hai (jo **0.0 se 10.0** tak hota hai). 10.0 ka matlab sab se zyada khatarnak (Critical).

Yeh score 3 baray factors par banta hai:
 * **Impact (Asar):** Is se kitna nuksan ho sakta hai? Kya data chori ho sakta hai ya server band ho sakta hai?
 * **Complexity (Mushkilat):** Isko exploit karna kitna aasan ya mushkil hai? Kya aik click se exploit ho jata hai ya bohot papad belne parhte hain?
 * **Availability (Dastiyabi):** Kya iska exploit publically available hai aur koi bhi isko aaram se use kar sakta hai?
> **Companies Iska Kya Karti Hain?** Companies ke paas hazaron servers hote hain. Woh CVSS score dekh kar faisla karti hain ke sab se pehle kis bug ko theek (patch) karna hai. Jo 9.8 ya 10.0 score wale hote hain, unhe foran theek kiya jata hai.
> 
### 3. Exploit-DB aur PoC (Proof of Concept)
Jab CVE register hoti hai, toh cyber security researchers us vulnerability ko prove karne ke liye aik chota sa script ya code likhte hain jise **PoC (Proof of Concept)** kehte hain. **Exploit-DB** jaisi websites par in CVE IDs ke sath unke PoC scripts pade hote hain, jinhe as a Red Teamer ya Pentester aap download kar ke target par test kar sakte hain.
### Ab Practical Kaise Karna Hai?
Is task ko solve karne ke liye aapko yeh steps follow karne hain:

 1. TryHackMe ke page par neeche **"Show Site"** (ya View Site) ka button hoga, us par click karein. Aik split-screen khulegi jo unki fake *Vulnerability Database* hogi.

 2. Us database ke search bar mein aapne search karna hai: **CVE-2026-1337**.

 3. Search karne ke baad us vulnerability ki details khulengi.

 4. Wahan par aapko check karna hai ke uski **CVSS Scoring** kya di gayi hai (jaise koi number hoga 7.5, 9.8, wagera) ya phir koi specific detail jo neeche diye gaye question mein puchi gayi ho.

## After Visiting Website 

### 1. Main Details & Description
 * **CVE ID:** CVE-2026-1337 (Severity rating upar **CRITICAL** di gayi hai).
 * **Vulnerability Type:** Yeh aik **SQL Injection** vulnerability hai (CWE-89).
 * **Kahan Par Hai?:** Apache WebPortal ke versions 3.2.0 se lekar 3.4.5 tak mein hai, jahan attacker login module ke "userid" parameter ke zariye arbitrary SQL commands run kar sakta hai.

### 2. CVSS Score & Metrics (Aapke Questions Ka Jawab)
Agar aapke task mein niche CVSS score ya metrics puchi hain, toh unka jawab yeh raha:
 * **CVSS Score:** **10 / 10** (Lekin maze ki baat yeh hai ke sath mein text label par ghalti se "HIGH" likha hua hai, jabki score 10 hamesha *Critical* hota hai—yeh simulation ka aik chota sa bug ya check ho sakta hai).

 * **Attack Vector (AV):** **NETWORK** (Iska matlab hacker bina physical access ke, internet ke zariye door baith kar attack kar sakta hai).

 * **Attack Complexity:** **LOW** (Isko exploit karna bohot aasan hai).
 * **Authentication:** **NONE** (Hacker ko attack karne ke liye kisi login account ya password ki zaroorat nahi hai).

 * **Impacts (Confidentiality, Integrity, Availability):** Teeno par **COMPLETE** impact hai (Iska matlab hacker poora database urha sakta hai, data badal sakta hai, aur system ko band kar sakta hai).

### 3. Solution (Fix)
 * Agar question mein pucha jaye ke isko theek kaise karna hai, toh solution hai: **Update to Apache WebPortal version 3.5.0 or later**, aur code mein prepared statements aur parameterized queries use karna.

### Some Important Details

Chalein isko break down karte hain:
### 1.CVE Ka Asal Kaam Kya Hai?
*"Website par test karne se pehle main Exploit-DB ya CVE par jaon ga yeh dekhne ke liye ke sab se dangerous vulnerabilities kaun si chal rahi hain."*
Real penetration testing mein process thoda ulta hota hai:

 1. Aap pehle CVE database nahi dekhte. Aap pehle **apne target (website) ko scan karte hain** (Reconnaissance).

 2. Scan kar ke aap pata lagate hain ke target website kaun sa software aur kaun sa **exact version** use kar rahi hai (e.g., aap ko pata chala ke target website Apache WebPortal v3.3.0 use kar rahi hai).

 3. **Ab aap CVE aur Exploit-DB par jayenge** aur wahan search karenge: *"Apache WebPortal 3.3.0"*. Phir CVE aap ko batayega ke *"Haan, is version mein CVE-2026-1337 maujood hai aur yeh 10/10 dangerous hai."*

> 💡 **Fark Samjhein:** CVE koi trending list nahi hai jo aap ko bataye ke aaj kal dunya mein kya chal raha hai. Yeh aik **searchable dictionary** hai. Jab aap ko target ka version mil jata hai, tab aap is dictionary mein aa kar us version ki bimari (vulnerability) dhoondte hain.
> 
### 2. Dusra Point: Apache WebPortal aur Server Ka Fark
Agar version 3.2.0 se 3.4.5 ke darmiyan hai toh website par khatra hai. Bas yaad rakhein ke **Apache HTTP Server** (jo pure website chalata hai) aur **Apache WebPortal** (jo aik specific application/software hai) do alag cheezein ho sakti hain.
