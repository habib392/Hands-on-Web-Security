Yeh task pure cyber security industry ke sab se popular platform **GitHub** ke baare mein hai. As a Red Teamer ya Penetration Tester, GitHub aapka sab se bada dost aur sathi hai, kyunke dunya bhar ke security researchers apne tools, scripts, aur exploits yahin publish karte hain.

Chalein isko details mein samajhte hain ke is task mein kya bataya gaya hai:

### 1. GitHub as an Exploit Resource (PoC Dhundna)
Jab dunya mein koi nayi vulnerability (CVE) aati hai, toh official databases (jaise NVD) par uski technical details aane mein thoda waqt lagta hai. Lekin GitHub par researchers bohot jaldi us vulnerability ka **PoC (Proof of Concept)** exploit script bana kar public kar dete hain.
 * **Security Use:** Agar aapko kisi pentest ke doran pata chala ke target par CVE-2026-1337 chal raha hai, toh aap seedha GitHub par ja kar search bar mein likhenge CVE-2026-1337 PoC. Aapko aisi repositories mil jayengi jahan python, bash, ya go language mein bane-banaye scripts pade honge jinhe run kar ke aap website par attack test kar sakte hain.

### 2. Sab se Badi Warning: Malicious PoCs ⚠️
Task mein aik bohot important warning di gayi hai jo har hacker ko pata honi chahiye: **GitHub par maujood har script par aankh band kar ke yakeen mat karein!**
 * Kuch log fake PoC banate hain. Woh upar likhte hain ke *"Yeh website ko hack karne ka script hai"*, lekin jab aap us script ko apne Kali Linux terminal par run karte hain, toh woh target ko hack karne ke bajaye **aapke hi computer ko hack kar leta hai** (backdoor execute kar deta hai).
 * **Rule:** Kisi bhi script ko run karne se pehle hamesha uske code ko khud khol kar parhein (read information) ke woh code aakhir kar kya raha hai.

### 3. README File Kya Hoti Hai?
GitHub repository ke andar aik file hoti hai jiska naam hota hai README.md. Yeh us project ki main guide ya manual hota hai jisme researcher likhta hai ke:
 * Yeh vulnerability kya hai?
 * Is script ko run kaise karna hai (usage)?
 * Iske liye kya requirements hain?
### Ab Practical Kaise Karna Hai?
Is final practical ko solve karne ke liye aapko yeh steps follow karne hain:

 1. TryHackMe ke page par neeche **"View Site"** ya **"Show Site"** ke blue button par click karein. Aik split-screen khulegi jo GitHub ki aik fake *Simulation* hogi.

 2. Wahan aapko CVE-2026-1337 ki aik repository (folder) dikhayi degi.

 3. Us repository ke andar jo **README** section (text guide) niche likha hoga, usko dhyan se parhein.

 4. Us README file ke andar hi aapko us sawal ka jawab mil jayega jo TryHackMe ne is task ke niche pucha hai.
